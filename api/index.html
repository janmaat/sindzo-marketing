API & integratie

FHIR R4 met Zib-profielen.

Coheza levert een open, gedocumenteerde API rondom de Nederlandse Zib-standaarden. Endpoints draaien in productie; publieke sandbox en interactieve docs zijn in afronding. Tot dan: spec en sandbox-toegang op aanvraag binnen 1 werkdag.

Publieke sandbox in afronding

api.coheza.nl/docs (Swagger UI) en de publieke sandbox-tenant gaan live in Q3 2026. Tot dan: stuur een mail naar api@coheza.nl — u krijgt de OpenAPI 3.1-spec, een test-client_id/secret en een gesegregeerde sandbox-tenant met synthetische cliëntdata. Reactie binnen 1 werkdag, geen NDA nodig.

FHIR R4 resources

Resources mappen naar Nederlandse Zorginformatiebouwstenen (Zib's) waar deze gedefinieerd zijn. Status reflecteert publieke endpoint-beschikbaarheid; interne implementatie kan verder zijn.

  • Patient
    Cliëntdemografie, BSN, contactgegevens — Zib-profiel.
  • Encounter
    Zorgmoment/contact — episode of care-context.
  • Observation
    Metingen, scores, vitale waarden.
  • MedicationStatement
    Actuele medicatie-overzicht (Zib MedicationOverview).
  • Condition
    Probleem-/diagnoseregistratie.
  • AllergyIntolerance
    Allergieën en intoleranties.
  • RelatedPerson
    Mantelzorger, wettelijk vertegenwoordiger, contactpersonen.
  • CarePlan
    Zorgplan-export — Omaha gemapped.
  • Practitioner / PractitionerRole
    Zorgverlener + AGB-koppeling.

Authenticatie & autorisatie

  • • OAuth2 client-credentials voor server-to-server (back-office, ketenpartners).
  • • OAuth2 authorization-code + PKCE voor cliënt-gerichte apps via cliëntportaal-flow.
  • • SMART-on-FHIR scopes (`patient/*.read`, `user/*.read`) voor fijnmazige toegang per resource.
  • • UZI-mTLS beschikbaar voor zorgverlener-authenticatie (in afronding, CIBG-cert vereist).

Wat we niet doen

Geen proprietary "Coheza-formaat" parallel aan FHIR. Geen lock-in via custom payloads. Geen rate-limit-tarieven voor klanten of geverifieerde ketenpartners. Geen verschil tussen wat in onze UI beschikbaar is en wat via de API kan — op een paar admin-endpoints na die expliciet gedocumenteerd zijn.
